1Goals & non-goals

Goals

  • Reduce active button components from 25 → 13.
  • Eliminate the five different corner radii currently used for filled CTAs.
  • Collapse three near-duplicate circle icon buttons (32 / 34 / 36pt) into one component with size variants.
  • Stop shipping feature-local buttons that are clones of DS components with one prop changed.
  • Keep every real visual distinction the product actually relies on — the green AI-coach affordance, the FAB, the Apple HIG button, the GymMode HUD prominence.

Non-goals

  • Visual redesign. Pixel output should be the same or close to it after migration.
  • Touching color tokens, type ramp, or the rest of the design system.
  • Optimizing GymMode HUD interactions — that's a separate project.
Guiding principle: if two buttons differ only in corner radius or height, they are the same button with a different prop. If they differ in role (FAB vs in-flow CTA), they stay separate.

2The v2 inventory

Thirteen components, grouped by role.

FGPrimaryButton

The filled CTA. Absorbs Auth, Gym, BugReportSubmit, DeleteAccountConfirm via shape + size + style props.

FGSecondaryButton

The outlined CTA. Absorbs the manual-build variant via tint="accent".

FGTextButton

Link-style. Absorbs FGTertiaryButton entirely — same behavior, different prop defaults.

FGSocialButton

Auth provider button. provider="apple" renders the HIG-compliant black variant; absorbs AppleSignInButton.

FGCircleIconButton

Single icon-in-circle component. Absorbs FGCircleButton, GymPauseCircle, RestPlayPauseCircle, GymNavArrow via size (sm/md/lg) and tint (neutral/elevated/accent/ghost).

FGCloseChevronButton

Kept as-is. Stroke-bordered dismiss is structurally different from the icon button family.

FGDashedAddRow

Kept as-is. Unique "in-panel add" pattern.

FGSelectableTile

NEW. Absorbs AutoProgressionButton, DaysPerWeekButton, EditorChip via size (chip/pill/tile) and activeTint (accent/info).

FGFloatingActionButton

Renamed from CreateProgramButton. Distinct FAB role — surface-anchored, not in-flow.

GuideMeButton

Kept as-is. The only green CTA in the app; anchors AI-coach identity.

ProfileRowButton

Kept as-is. Settings-row pattern. May be promoted to DS later if reused.

ColorSwatchButton

Kept as-is. One-off color-pick affordance.

RestTimerPill

Kept as a compound component. Its parts (RestAdjustCapsule, RestPlayPauseCircle) become internal — never used standalone.

—

Buttons that were promoted to internal-only or merged away aren't listed here. See the mapping table.

4Component specs

Prop surface and visual rules for each v2 component. Existing per-pixel specs (heights, radii, padding) carry over unchanged unless noted.

FGPrimaryButton

FGPrimaryButton(
  title:                String,
  style:                "accent" | "info" | "destructive"      // default: accent
  shape:                "capsule" | "rounded"                  // default: capsule  (rounded = RR md)
  size:                 "regular" | "tall"                     // default: regular (54) | tall (80)
  emphasis:             "normal" | "brand"                     // brand adds accent glow + UPPERCASE — onboarding only
  leadingSystemImage:   String?
  trailingSystemImage:  String?
  isLoading:            Bool
  disabled:             Bool
)

Replaces: FGPrimaryButton, FGAuthPrimaryButton, FGGymActionButton, BugReportSubmitButton, DeleteAccountConfirmButton.

FGSecondaryButton

FGSecondaryButton(
  title:                String,
  tint:                 "neutral" | "accent"                   // default: neutral
  shape:                "capsule" | "rounded"                  // default: capsule
  leadingSystemImage:   String?
  disabled:             Bool
)

FGTextButton

FGTextButton(
  title:                String,
  color:                "accent" | "primary" | "secondary" | "destructive"    // default: accent
  weight:               "regular" | "medium" | "semibold"      // default: regular
  size:                 "regular" | "large"                    // 14pt | 15pt
  disabled:             Bool
)

Replaces: FGTextButton, FGTertiaryButton. Tertiary's emphasizes prop is gone — express it as weight="medium".

FGSocialButton

FGSocialButton(
  title:                String?                                // auto-set per provider when nil
  provider:             "apple" | "google" | "generic"         // apple → black HIG variant
  systemImage:          String?                                // only used when provider="generic"
)

When provider="apple": background #000, white text, "Sign in with Apple", apple.logo. Apple HIG-locked.

FGCircleIconButton

FGCircleIconButton(
  systemImage:          String,
  size:                 "sm" | "md" | "lg"                     // 32 | 36 | 56
  tint:                 "neutral" | "elevated" | "accent" | "ghost"           // neutral = 8% fill · elevated = surfaceCard + shadow · ghost = transparent
  glyphTint:            "primary" | "secondary" | "accent"     // optional override
  disabled:             Bool
)

Replaces: FGCircleIconButton, FGCircleButton, GymPauseCircle, RestPlayPauseCircle, GymNavArrow.

FGSelectableTile new

FGSelectableTile(
  label:                String,
  selected:             Bool,
  size:                 "chip" | "pill" | "tile"               // RR8 small · capsule · RR12 56pt
  activeTint:           "accent" | "info"                      // fill color when selected
  monospace:            Bool                                   // digit alignment for editor use
  leadingSystemImage:   String?
)

Replaces: AutoProgressionButton, DaysPerWeekButton, EditorChip.

FGFloatingActionButton

FGFloatingActionButton(
  systemImage:          String,                                // usually "plus"
  style:                "accent" | "neutral"                   // default: accent
)

54×54 circle, drop shadow, surface-anchored. Replaces CreateProgramButton.

Unchanged (no spec changes)

FGCloseChevronButton, FGDashedAddRow, GuideMeButton, ProfileRowButton, ColorSwatchButton, RestTimerPill.

5Migration plan

Phase 0
Freeze & inventory · ½ day

Lock the current set; no new buttons added during migration. Build the screen × component matrix (which screens use which buttons). This is the test suite — every cell must still render correctly post-migration.

Phase 1
This doc · 1 day

Spec + mapping (this document). Sign-off required before code changes start. Design review + one engineering review.

Phase 2
Build v2 alongside v1 · 2–3 days

Add new components to DesignSystem/Components/ without removing the old ones. Tag v1 components with @available(*, deprecated, message: "Use FGPrimaryButton(shape: .rounded, size: .tall)"). Both versions ship; no screens change yet.

Phase 3
Migrate by surface · 1–2 weeks (parallelizable)

One PR per screen, not per component. Order by risk:

  1. Bug Report screen (2 buttons, low traffic)
  2. Profile settings rows (1 button)
  3. Delete Account confirm (1 button)
  4. Color picker (static)
  5. Programs list FAB (1 button)
  6. Create Program flow — days-per-week, manual-build (2 buttons)
  7. Onboarding — Apple/Google, Auth primary (3 buttons, sensitive flow)
  8. Workout logging — EditorChip, AutoProgression (2 buttons)
  9. GymMode HUD — pauseCircle, nav arrows, RestTimerPill (highest risk, full dogfood week before merge)
Phase 4
Delete v1 · ½ day

When git grep FGAuthPrimaryButton returns zero, delete it. Same for all merged components. Deprecation warnings make this mechanical.

Phase 5
Update canvas + docs · ½ day

Replace this button review page with the v2 inventory. Archive the v1 canvas as Buttons-v1-archive.html. Update onboarding docs for new engineers.

Guardrails

  • One owner. One designer + one engineer pair runs all the migration PRs. Distributed consolidation drifts.
  • Snapshot-test before starting. Screenshot every screen in light + dark. Diff after each PR. Catches "I forgot Gym used RR14 not capsule."
  • No scope creep. Someone will ask to redesign the rest timer "while we're in there." No. This is mechanical refactor; visual redesigns are separate projects.
  • GymMode is last and slow. Power users have muscle memory for the HUD. Migrate it last and dogfood for a full week before shipping.

Realistic timeline

About 3–4 weeks as focused work, or a steady 2-month background project if shipping features in parallel. The build is the short pole; the migration is the long one.

6Open questions for review

  1. Q1 — FGCircleButton (32pt) → sm or md? resolved Clean path. Drop the 32pt size entirely. Migrate Program Editor's top bar to size="md" (36pt) to match every other top bar. FGCircleIconButton's size prop becomes "md" | "lg" only.
  2. Q2 — Keep the Auth glow + UPPERCASE? resolved Drop it. Onboarding uses plain FGPrimaryButton shape="rounded" size="tall". The emphasis prop is removed from the FGPrimaryButton spec.
  3. Q3 — Promote ProfileRowButton + FGFloatingActionButton to DS now? resolved Now. Both move into DesignSystem/Components/ as part of Phase 2.
  4. Q4 — FGSelectableTile sizes: 3 or 2? resolved 3 sizes for now (chip, pill, tile). Re-evaluate after 3 months in production — if chip and pill end up identical in practice, collapse then.
  5. Q5 — GuideMeButton in DS or feature-local? resolved Promote to DS. Anticipating AI-coach affordances across multiple surfaces.
Spec impact of resolutions:
  • FGPrimaryButton drops the emphasis prop entirely.
  • FGCircleIconButton's size = "md" | "lg" only (was sm/md/lg).
  • FGFloatingActionButton, ProfileRowButton, GuideMeButton all live in DesignSystem/Components/.
  • Program Editor top bar gets a 4pt size bump as part of Phase 3 migration.
